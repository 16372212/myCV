jvm 入口

在JDK的源代码中,编译后的 java 可执行文件的入口点位于 src/java.base/share/native/libjli/java.c 文件中的 JavaMain和JLILaunch 函数

在我们着手调试JVM内存分配的代码前先堆内存分配进行一个简单的了解

内存部分的结构：

内存分配：

当新new一个对象的时候，首先执行 (虚拟机视角) 1.类加载检查 -> 2.分配内存 -> 3.内存空间初始化为0值 -> 4.对对象进行必要的设置（将一些信息存放在对象的对象头Object Header) -> (Java程序视角) 5.执行构造函数

在HotSpot JVM中,内存分配主要发生在两个地方:Java堆和本地(C++)堆。

1. Java堆中的分配

Java堆:

存储Java对象和数组。
由JVM管理,受垃圾收集的控制。
Java应用程序可以直接与之交互。
它的大小可以在JVM启动时通过-Xms和-Xmx参数来设置。

Java堆的分配入口函数位于 src/hotspot/share/gc/shared/collectedHeap.hpp 文件中。CollectedHeap 是一个抽象类,代表了Java堆,不同的垃圾收集器会提供不同的实现。在 CollectedHeap 类中,有两个重要的内存分配函数:

  • allocate_new_tlab(size_t min_size, size_t requested_size, size_t* actual_size): 这个函数用于为线程分配一个新的线程局部分配缓冲区(TLAB)。TLAB是为了提高内存分配的效率,每个线程会从自己的TLAB中分配小的对象。

  • mem_allocate(size_t size, bool* gc_overhead_limit_was_exceeded): 这个函数用于在Java堆上分配一个指定大小的内存块。如果TLAB被耗尽,或者要分配的对象太大,就会直接调用这个函数。

2. 本地(C++)堆的分配

C++堆:

存储JVM内部数据结构,如类数据结构(如 Klass),线程数据结构,JIT编译器生成的代码等。
由C++运行时库管理,不受JVM垃圾收集的控制。
Java应用程序通常不会直接与之交互。
它的大小取决于操作系统和C++运行时库的设置。

入口点是 src/hotspot/share/memory/allocation.hpp 中的函数:

  • AllocateHeap(size_t size, MEMFLAGS flags): 在C++堆上分配一个指定大小的内存块。
  • ReallocateHeap(void* old, size_t size, MEMFLAGS flag): 在C++堆上重新分配一个内存块。

这些函数最终会调用操作系统的内存分配函数,如 malloc 和 realloc。

native方法是什么？

native方法确实可以被看作是Java和JVM的本地代码(通常是C或C++)之间的一种接口或者桥梁。

带了native关键字的，说明Java的范围调用不到了，就回去调用底层c语言的库。此时会进入本地方法栈。然后会调用本地方法本地接口JNI。（本地方发栈存储native方法的调用状态。）

当Java代码调用一个native方法时,它实际上是在请求JVM执行这个方法的本地实现。JVM通过JNI(Java Native Interface)找到并调用相应的C或C++函数。
