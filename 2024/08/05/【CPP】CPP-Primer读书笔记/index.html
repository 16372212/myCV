chapter 1

编译

编译可以使用g++或者微软的c1

g++ -o prog1 prog1.cc

命名空间std避免名字定义的冲突。使用作用域运算符：：指明运算空间。

来自标准库的头文件使用<>，否则使用“”包围。

头文件的使用

头文件（.h 文件）在C++中并不完全等同于Java中的接口，但它们确实扮演了类似的角色，即声明接口，使其他文件可以知道类和函数的定义，而不需要知道具体的实现细节。

•	头文件：包含类、函数和变量的声明。它告诉编译器这些类和函数是什么样的，如何调用它们，但不包含具体的实现细节。头文件通常以 .h 或 .hpp 结尾。
•	实现文件（.cpp 文件）：包含类和函数的具体实现。它们实现了头文件中声明的所有函数和方法。

Sales_item.cc, Sales_item.h 以及执行main函数的main.cc

chapter2

变量的定义

习惯来讲，对象是具有某种数据类型的内存空间。

初始化

初始化！=赋值

初始化是创建的时候赋予的初始值，赋值是把对象的当前值擦除，用新的值代替。

cpp 11可以支持花括号初始化变量。但是列表初始化有丢失信息的风险时，编译器报错。

默认初始化：

变量类型决定默认值和定义变量的位置。

内置类型的变量如果没有显示初始化，值由定义的位置决定。定义于函数体之外的变量不被初始化。

每个类各自决定初始化对象的方式。大多数都支持无需显示初始化而定义对象。

  • string 默认空串。

未初始化的变量有一个不确定的值，很难调试。大部分编译器能给出警告，但有的不给。

内置类型指的是编译器直接支持的类型，不需要包括头文件就能使用。也叫做基本类型。

变量声明和定义的关系

声明和定义区分开。

  • 变量声明：只是说明了类型和名字，如果有初始化就成了定义了。 extern int i;
  • 定义：创建与名字关联的实体。（只能一次）

作用域

嵌套的作用域都能访问被嵌套作用域的名字。同时，内层作用域可以重新定义外层作用域已有的名字。

复合类型

引用

现在我们说的引用指的是左值引用（可以取地址的对象就是左值，不可取的是负值）

引用必须被初始化
写成&d的格式。
引用只能绑定在对象上。
只能初始化一次。

int ival = 10;
int &d = ival;

可以理解为别名。

指针

和引用的区别：

  • 指针本身就是对象，允许赋值和拷贝、
  • 指针不需要在定义的时候赋值。

存放某个对象的地址。

有两种写法：

  • 修饰符和变量标识写在一起，int *p, *q;
  • int* q;
