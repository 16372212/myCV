Service Mesh 服务网络架构

是一种架构模式，专门用于微服务通讯。服务网格通过服务代理来处理微服务间的通信管理，包括流量管理、安全认证和监控等，从而将这些复杂性从应用程序中解耦出来。

Service Mesh can be defined as an infrastructure layer which handles the inter-service communication in a Microservice architecture. Service Mesh reduces the complexity associated with a Microservice architecture and provides lot of the functionalities like

  • Load Balancing \ - Service Discovery - Health Check \ - Authentication \ - Traffic Management & Routing - Circuit Breaking and Failover Policy \ - Security \ - Metrics & Telemetry - Fault Injection

服务网格service mesh中有数据平面和控制平面的概念：

  • 数据平面（Data Plane）：由一组 Sidecar 代理组成，这些代理与每个微服务一起部署，处理服务间的通信。（流量管理、安全、可观察性）部署方式：通常作为 Sidecar 容器，与每个微服务实例一起运行在同一个 Pod 中。

  • 控制平面（Control Plane）：管理数据平面的代理，提供全局的配置、策略和服务发现功能。

Sidecar模式: implement service mesh

To implement the service mesh, you can deploy a proxy alongside with your services. This is also known as the Sidecar Pattern.

  • Sidecar 是一种设计模式，用于分离微服务中的辅助功能。
  • Sidecar 与主应用服务部署在同一个环境中，如同一个容器 Pod 中，使得它们能够高效地进行通信。

service mesh和sidecar的关系

和service mesh的关系：Service Mesh 利用了 Sidecar 模式，通过在每个微服务旁边部署 Sidecar 代理，实现了对服务间通信的集中管理和控制。
