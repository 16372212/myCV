入口

首先找到一个入口：org.apache.ibatis.executor.BaseExecutor的doQuery方法。

具体步骤

step 1

我们进入这个doQuery方法里的Query函数中去，该函数创建了一个 StatementHandler 实例。StatementHandler 是 MyBatis 用于处理 SQL 语句的核心组件之一,负责准备、执行和返回查询结果。

在 Mybatis 源码中,主要有 4 个 StatementHandler 实现类:

  1. SimpleStatementHandler
    用于执行简单的 SQL 语句,如 SELECT, UPDATE, DELETE 等。它直接使用 JDBC 的 Statement 对象来执行 SQL 语句。适用于不需要参数绑定的情况。

  2. PreparedStatementHandler

    用于执行包含占位符的 SQL 语句,如 SELECT * FROM users WHERE id = ?。使用 JDBC 的 PreparedStatement 对象来执行 SQL 语句,可以绑定参数。

  3. CallableStatementHandler
    用于执行存储过程或函数的 SQL 语句,如 CALL my_procedure(?, ?)。
    它使用 JDBC 的 CallableStatement 对象来执行 SQL 语句,可以绑定参数和处理返回结果。
    适用于需要执行存储过程或函数的情况。

  4. RoutingStatementHandler
    一个路由类,根据 SQL 语句的类型,委托给上述 3 种 StatementHandler 之一

step 3

接下来我们进入handleResultSet方法看看，这个函数是结果匹配对象的核心函数之一：
它包含两个handle函数，一个是handleRowValuesForSimpleResultMap一个是handleRowValuesForNestedResultMap。handleRowValuesForSimpleResultMap 是用于处理简单的 ResultMap 定义,即没有嵌套查询的情况。它直接从结果集中提取行数据,并创建相应的结果对象。

handleRowValuesForNestedResultMap 可能需要处理多个相关的结果集,以满足嵌套 ResultMap 的需求。需要先创建父对象,然后根据嵌套的 ResultMap 定义,递归地创建子对象,并建立父子对象之间的关联关系。

这次我们只涉及到了handleRowValuesForSimpleResultMap.它根据简单的 ResultMap 定义,从 ResultSet 中提取行数据,并创建相应的结果对象。

下面是这个方法的具体实现逻辑:

  • 创建 DefaultResultContext 对象,用于保存处理过程中的上下文信息。

  • 获取当前的 ResultSet 对象。（resultSet里记录了sql以及结果的行数、rawData等信息，可以查看查询结果的rawData

  • 根据给定的 RowBounds 对象,跳过指定数量的行。

  • 开始遍历 ResultSet，直到达到 RowBounds 定义的范围。

  • 对于每一行数据:

    • 通过 resolveDiscriminatedResultMap 方法解析出正确的 ResultMap。
    • 调用 getRowValue 方法从当前行中获取对应的结果对象。
    • 将获取的结果对象存储到 resultHandler 中。
    • 处理完所有行数据后,返回。

接着我们进入getRowValue方法：

rowValue = getRowValue(rsw, discriminatedResultMap, rowKey, null, partialObject);

step 4

这个函数主要是MyBatis 中用于从结果集中获取行数据,并创建相应的结果对象。它调用 createResultObject 方法创建结果对象实例。这个方法会根据 ResultMap 的定义,使用构造方法或工厂方法来创建结果对象。

如果根据ResultMap创建的结果对象不为空,且没有为该类型注册类型处理器,则进行以下操作:

  1. 创建结果对象的 MetaObject。

  2. 判断是否应该应用自动映射。如果应该,则调用 applyAutomaticMappings 方法执行自动映射。(当创建了结果对象实例,且没有为该类型注册类型处理器时,会判断是否应该应用自动映射,如果 autoMapping 属性被显式地设置为 false，或者当前 ResultMap 是嵌套的 ResultMap，则不应用自动映射。同时,它也会递归地检查父 ResultMap 的配置,以确保自动映射的一致性。)

  3. 调用 applyPropertyMappings 方法,将结果集中的数据映射到结果对象的属性上。

  • 检查是否存在延迟加载相关的属性。

好家伙这才是核心，赶紧点进去看看。
